# Presentia — Estado del proyecto v1.0
**Fecha:** 2026-03-17
**Stack:** Django 5.1 + DRF + MySQL · React 19 + Vite + Tailwind v4

---

## Módulos completados

### Autenticación y roles
- Registro e inicio de sesión con tokens JWT (en memoria, sin localStorage)
- Roles diferenciados: `admin` y `maestro`
- Rutas protegidas por rol en frontend y backend
- Panel de administración para gestión de maestros (solo visible para `admin`)
- Logout con limpieza de tokens

### Grupos (Classrooms)
- Creación, edición y listado de grupos
- Alta y edición de alumnos con matrícula
- Captura de muestras faciales por alumno (mínimo 5 fotos)
- Indicador visual de alumnos con/sin muestras suficientes
- Las imágenes se almacenan como arrays numpy 128×128 en escala de grises (pickle) en base de datos

### Toma de asistencia
- Creación de sesiones de asistencia por grupo y fecha
- Subida de video por sesión (MP4, AVI, MOV, MKV)
- Procesamiento automático en background (threading daemon):
  - Entrenamiento de reconocedor LBPH (`opencv-contrib-python`) con las muestras del grupo
  - Detección de rostros por frame con Haarcascade
  - Match por confianza LBPH (umbral < 100)
  - Alumno marcado como **presente** si aparece en ≥ 10% de los frames procesados
  - Video eliminado automáticamente tras el procesamiento
- Interfaz reorganizada por grupos:
  - `/attendance` → cards por grupo (nombre, última sesión, conteo)
  - `/attendance/classroom/:id` → sesiones del grupo ordenadas por fecha
  - `/attendance/:id` → detalle de sesión con lista presente/ausente, barra de porcentaje y botón de reporte
- Reportes HTML de asistencia generados desde Django templates (acceso directo sin JWT)
